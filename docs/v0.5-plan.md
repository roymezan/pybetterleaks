# v0.5 Plan

v0.5 is a release-hardening pass. The user-facing scanner API is already broad
enough for alpha users, so this release focuses on making every tag easier to
trust, inspect, and explain.

## Goals

- Bump the Python package version to `0.5.0`.
- Inspect every wheel artifact before publishing.
- Reject accidental universal wheels and unsupported musllinux wheels.
- Verify that every release wheel contains exactly one packaged native library
  and `py.typed`.
- Verify release checksums after generating them.
- Build the native bridge in the Python CI job before coverage so native smoke
  tests are included in the coverage gate.
- Publish GitHub release notes from tags and attach `SHA256SUMS`.
- Add GitHub artifact attestations for wheels and checksums.
- Document the release supply-chain story.
- Document the deprecation policy before the API approaches `1.0.0`.

## Non-Goals

- No new scanner modes.
- No public API reshaping.
- No musllinux/Alpine wheels.
- No sdists.
- No automated SBOM generation yet.
- No artifact signing beyond GitHub artifact attestations.

## Implementation

- Add `scripts/inspect_wheels.py` for artifact checks that can run locally and
  in GitHub Actions.
- Extend `scripts/checksums.py` with `--verify`.
- Add release-tool unit tests with synthetic wheel artifacts.
- Add native bridge build setup to the Python CI job before coverage.
- Run wheel inspection in `wheels.yml` after `cibuildwheel`.
- Run wheel inspection, checksum generation, checksum verification,
  provenance attestation, PyPI publishing, PyPI smoke, and GitHub release
  creation in `publish.yml`.
- Add `.github/release.yml` so generated release notes are grouped by label.
- Add docs for supply chain, provenance, checksums, and deprecation policy.

## Acceptance Criteria

- `uv run coverage run -m pytest`
- `uv run coverage report`
- `uv run ruff check .`
- `uv run mypy python`
- `uv run --group docs mkdocs build --strict`
- A locally built wheel passes `scripts/inspect_wheels.py`.
- A generated checksum file passes `scripts/checksums.py --verify`.
- The release checklist explains the new tag-to-PyPI flow.
